# 数据版本管理功能设计

> 当前项目已具备内容去重缓存（ZIP hash + 单图 embedding cache），
> 但缺少显式的版本概念。本文档描述如何在此基础上实现数据集版本管理。

## 当前已有的增量基础

| 机制 | 实现 | 效果 |
|------|------|------|
| ZIP 内容缓存 | `_cache/{sha256}/dataset.json` | 同一 ZIP 重复上传秒级命中，跳过解析+推理 |
| 单图 embedding 缓存 | Milvus `embedding_cache`（key=文件 SHA-256） | 同一张图出现在不同数据集中，只推理一次 |
| 两阶段上传 | `create_dataset_without_embedding` → 先展示 → `complete_dataset_embedding` → 后台推理 | 新增图片先出预览 |

本质上已支持**无变更跳过**：图片变了（hash 不同）才重新推理，没变直接复用缓存。

## 目标

在上传时支持指定基础版本，自动对比新旧差异，只对新图/变图推理（数量可控），形成可追溯、可切换的版本链。

## 版本数据模型

在 `_cache/{dataset_name}/` 下维护版本索引文件 `versions.json`：

```json
{
  "name": "my-dataset",
  "versions": [
    {
      "version_id": "v3",
      "dataset_id": "uploaded-1719312000-my-dataset",
      "parent_version_id": "v2",
      "version_number": 3,
      "diff_summary": {
        "added": 120,
        "removed": 15,
        "unchanged": 4865,
        "modified": 8
      },
      "image_count": 4993,
      "created_at": "2026-07-01T10:30:00Z"
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `version_id` | string | 版本唯一标识，如 `v1`、`v2` |
| `dataset_id` | string | 对应现有 dataset_id |
| `parent_version_id` | string 或 null | 父版本 ID，首版为 null |
| `version_number` | int | 自增版本号 |
| `diff_summary` | object | 相对父版本的变更统计 |
| `image_count` | int | 该版本图片总数 |
| `created_at` | ISO string | 创建时间 |

## 上传流程改动

```
POST /api/dataset/upload  +  ?base_version_id=v2

  1. 解压 ZIP，遍历图片，每张计算 SHA-256
  2. 加载 base_version (v2) 的图片 hash 列表（从 versions.json 提取）
  3. diff 对比：
     ├─ hash 相同 → unchanged → 复用 embedding_cache
     ├─ hash 新增 → added → 只对新图做 BGE 推理
     ├─ hash 在 base 中存在但新版没有 → removed → 标记
     └─ 同文件名不同 hash → modified → 重新推理
  4. 生成新版本 payload，记录 parent_version_id + diff_summary
  5. 追加到 versions.json
```

**关键优化**：只对 `added + modified` 图片跑 BGE 推理，`unchanged` 图片直接复用存储的 embedding。对于 5000 张图新增 120 张的场景，推理量从 5000 降到 128。

## 新增 API

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/dataset/versions?name=my-dataset` | 列出某数据集所有版本 |
| GET | `/api/dataset/{id}/diff?base=v2&target=v3` | 两个版本间差异详情（图片列表） |
| POST | `/api/dataset/upload?base_version_id=v2` | 增量上传入口（已有接口加参数） |
| POST | `/api/dataset/switch?version_id=v3` | 切换当前展示版本 |

## 前端 UI 交互

### 版本选择器（topbar）

```
[my-dataset ▼] [v3 (latest) ▼]  [上传新版本]
```

- 数据集名称下拉：切换不同数据集
- 版本下拉：`v1` / `v2` / `v3 (latest)`，切换后重新加载对应 payload
- "上传新版本"按钮：弹出上传对话框，自动以当前最新版本为 base

### diff 视图（grid）

- 相对 parent 版本**新增**的图片 → 绿色边框 + "NEW" 标签
- 相对 parent 版本**删除**的图片 → 红色边框 + "REMOVED" 标签（仅在 diff 模式对比时显示）
- **修改**的图片 → 黄色标记 + "MODIFIED"

### 统计面板（sidebar）

diff 模式下显示：

```
版本变更: v2 → v3
  新增    120 张
  删除     15 张
  修改      8 张
  未变   4865 张
  总计   4993 张
```
