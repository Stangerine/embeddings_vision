# 数据清洗建议：离群与重复判断

> 清洗建议（离群样本、重复图片）在**数据集上传 embedding 完成后于后端预计算**，结果随
> `DatasetPayload.cleaning` 返回，前端直接读取秒出，不在浏览器跑 O(n²) 计算。

## 设计动机（为什么改）

旧实现存在两个问题：

1. **结果不合理**：5000 张图却报告离群 290、重复对 165573。根因：
   - 重复检测用 **2D PCA 投影坐标**做欧氏距离阈值 + O(n²) 全配对。2D 投影把高维语义差异压扁，密集区配对数平方膨胀。
   - 离群用 `mean+1.5σ` 固定切，正态分布天然约 6-7% 被标记，与“离群”语义不符。
2. **响应慢**：两个检测器都在浏览器跑 O(n²)，5000 点约 1250 万次开方，重复对 16 万条还要建 Set + canvas 画线。

**改进方向**：用 Milvus 中已存的 **768 维高维 embedding + 余弦相似度**做真正的语义判定，并在上传后台阶段**预计算**，不阻塞图片与散点可视化展示。

## 判定算法

实现位于 `backend/dataset_service.py` 的 `compute_cleaning(images, embeddings)`，纯 numpy。

前置：把所有向量 L2 归一化后做 `sim = M @ M.Tᵀ`（点积即余弦相似度），裁剪到 `[-1, 1]`。

### 离群样本（Outliers）
- 对每张图取其 **k 个最近邻**（默认 k=6，排除自身）的相似度。
- 离群分数 = `1 - mean(topk_sim)`，即到 k 近邻的**平均余弦距离**。距离越大越孤立。
- 取分数最大的 **(1 - outlier_percentile)** 比例为离群（默认 percentile=0.95 → 约 **5%**）。
- 按分数降序排序，截断到 `max_outliers`（默认 0=自动缩放为 `max(50, n×15%)`）。
- 输出：`outliers: [{ imageId, score }]`。

### 重复图片（Duplicates）—— 互最近邻（Mutual Nearest-Neighbor）

> 关键变更：**报告“重复图片数量”，不是“重复对数”**。

车辆数据集的 embedding 空间极度稠密（实测 cos≥0.98 的配对超过 190 万对），全配对阈值法会爆炸。
因此改用互最近邻：

- 对每张图 `i`，找它**相似度最高的邻居** `nn[i]`（排除自身）。
- 当 `i` 与 `nn[i]` **互为最近邻**（`nn[nn[i]] == i`）**且**相似度 `≥ dup_threshold`（默认 0.98）时，
  判定 `i` 是重复图片。
- 输出参与重复关系的**去重图片集合**（每张图最多出现一次），按邻居相似度降序，截断到 `max_pairs`（默认 0=自动缩放为 `max(50, n×30%)`）。
- 输出：`duplicates: [{ imageId, similarity }]`，`similarity` 为该图与其互最近邻的余弦相似度。

这样每张重复图片只计一次，数量上界为 n，语义清晰，不随稠密度爆炸。

## 配置参数

集中在 `config/dataset_settings.py` 的 `CleaningSettings`，可用环境变量覆盖：

| 参数 | 环境变量 | 默认值 | 说明 |
|------|----------|--------|------|
| `dup_threshold` | `CLEANING_DUP_THRESHOLD` | `0.98` | 重复判定的余弦相似度下限（互最近邻） |
| `outlier_percentile` | `CLEANING_OUTLIER_PERCENTILE` | `0.95` | 离群分位线，0.95 → 最远约 5% 为离群 |
| `k_neighbors` | `CLEANING_K_NEIGHBORS` | `6` | 离群计算的 k 近邻数 |
| `max_pairs` | `CLEANING_MAX_PAIRS` | `0`(自动) | 重复**图片**数量上限。0=自动缩放为 `max(50, n×30%)`；显式设置则用固定值 |
| `max_outliers` | `CLEANING_MAX_OUTLIERS` | `0`(自动) | 离群样本数量上限。0=自动缩放为 `max(50, n×15%)`；显式设置则用固定值 |

> **自动缩放**：`max_pairs` 和 `max_outliers` 默认 0，运行时根据数据集图片数 `n` 动态计算上限。
> 离群分位线（0.95）自然控制数量在 ~5%，15% 为保护上限；重复互最近邻最多 n 张，30% 为合理上限。
> 小数据集兜底最少 50 条。如需固定值，设置对应环境变量即可覆盖。

## 数据流

1. 上传分两步：`create_dataset_without_embedding`（先展示图片）→ `complete_dataset_embedding`（后台算 embedding+散点）。
2. `complete_dataset_embedding` 在 `apply_embedding_projection` 之后、embedding 为 `ready` 时调用
   `compute_cleaning`，结果挂到 `payload["cleaning"]`。embedding 失败回退路径不计算 cleaning（置空）。
3. 持久化：独立的 Milvus `cleaning` 集合（`dataset_id` PK + `cleaning_json`），
   通过 `MilvusStore.save_cleaning` / `load_cleaning` 读写；`_write_dataset` / `_read_dataset` 自动合并进 payload。
4. 前端：`DatasetPayload.cleaning` → Zustand store 的 `cleaning` 字段 →
   `useCleaningIssues` 直接读取（不再跑浏览器计算）。

## 前端类型

`src/lib/types.ts`：

```ts
export interface CleaningOutlier {
  imageId: string;
  score: number; // 到 k 近邻的平均余弦距离
}

export interface CleaningDuplicate {
  imageId: string;
  similarity: number; // 与互最近邻的余弦相似度
}

export interface CleaningSuggestions {
  outliers: CleaningOutlier[];
  duplicates: CleaningDuplicate[];
}
```

## 散点图叠加层（`src/components/scatter-view.tsx`）

- **离群点**：红色光环 ring（绘制在最上层）。
- **重复图片**：蓝色光环 ring（每张重复图片单独标记，不再画配对连线）。
- 计数标签：「离群 N」「重复图片 M」（旧的「重复对」已移除）。

## 性能与扩展

- 当前实现一次性构建 n×n 相似度矩阵。n≤8000 时（约 250MB float32）可接受，实测约 1 秒。
- 若未来数据集 >8000，需对 `M @ M.Tᵀ` 做分块计算（本次未实现，已在代码注释标注）。

## 验证结论

- 合成测试（100 基向量 + 10 近似克隆）：重复图片恰好检出 **20 张**（10 克隆 + 10 源，全部互最近邻），离群约 5.5%。
- `pnpm ts-check` / `pnpm lint:build` 通过；19 个后端单测通过。
